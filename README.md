# Optimize SLIC With Intel KNL Processors

中山大学超算队2022届优化组选拔题

## 我需要做什么？

在集群上运行脚本 `run.sh`，应该就能测试出基准性能。

优化源代码 `SLIC.cpp`，尽可能地提升程序在三个算例上的计算时间。你可能会尝试的优化方法：

* 编译器优化：尝试一些编译参数
* 线程级并行：OpenMP, Pthread
* 向量指令集：AVX系列指令集
* 程序访存优化
* 其他一切可能的办法

你可以查看 `OneOptimization.log` 参考某位队员优化代码后的运行结果。

## 如何提交？

选拔赛结束的时候，你需要在你的仓库中提交：

* 经过你优化的最终源代码
* 此代码在三个算例上的运行时间
* 你优化代码的过程记录，如果你采用了多种优化技巧，**请按顺序依次描述你的方法、记录实验过程和结果**，并提出自己的思考。

**你可能会尝试很多无效的改动，但你应该尝试它们，并且记录下实验过程和你的思考**

## 无从下手？

检查一下你是否：

1. 首先将未优化的代码在集群上成功运行，并记录初始性能
2. 通过阅读代码、搜索资料来理解算法：尝试理解每部分代码的功能或目的
3. 探索和思考：哪一部分代码耗费的时间最多？如何减少？
4. （*可选*）尝试使用Profiler解答上述问题：perf, Intel VTune Profiler
5. （*可选*）比较了解KNL处理器的架构特性？搜索相关资料，或阅读 [这里](https://github.com/SYSU-SCC/Recruit-22/tree/main/0-Tutorials) 提供的资料。
